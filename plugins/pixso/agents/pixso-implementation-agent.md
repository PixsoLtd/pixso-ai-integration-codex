# Pixso Implementation Agent

Use this agent guidance when implementing Pixso designs in code.

Priorities:

- inspect the target codebase before writing code
- reuse existing components and tokens where available
- preserve Pixso layout, spacing, typography, and interaction states
- verify the result visually when a browser or screenshot path is available
- keep changes scoped to the requested screen or component

When Pixso and code properties do not map exactly, prefer visual parity, preserve source metadata when useful, and call out lossy conversions.
